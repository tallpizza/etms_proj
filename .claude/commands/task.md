# PRD 기반 작업 목록 생성

## 목표

PRD의 상위 작업을 기반으로 에이전트가 실행할 수 있는 상세 하위 작업 목록을 생성하고, 병렬 처리 가능한 작업을 식별하는 AI 어시스턴트 가이드

## 입력

- PRD 파일 경로: `/prd/[feature_name].md`
- PRD 내 "🚀 구현 작업 순서 (상위 작업)" 섹션 활용

## 출력 형식

- **형식:** 마크다운 (`.md`)
- **위치:** `/tasks/`
- **파일명:** `tasks-[feature_name].md` (PRD 파일명과 동일하게 매칭)

## 실행 프로세스

### 1단계: PRD 분석

- 지정된 PRD 파일의 상위 작업 목록 읽기
- 각 상위 작업의 목표와 산출물 확인
- 작업 간 의존성 파악

### 2단계: 하위 작업 생성

- 각 상위 작업을 3-7개의 구체적인 하위 작업으로 분해
- 에이전트가 독립적으로 실행할 수 있는 단위로 작성
- 명확한 입력, 처리, 출력 정의

### 3단계: 병렬 처리 식별

- 작업 간 의존성 분석
- 동시 실행 가능한 작업들을 🔄 아이콘으로 표시
- 병렬 처리 시 주의사항 명시

### 4단계: 관련 파일 매핑

- 각 작업에서 생성/수정할 파일 명시
- 작업 간 파일 충돌 가능성 체크

### 5단계: 작업 목록 저장

생성된 문서를 `/tasks/` 디렉토리에 저장

## 출력 템플릿


## 병렬 처리 식별 기준

### 병렬 가능 조건

1. **파일 독립성**: 서로 다른 파일을 다루는 작업
2. **데이터 독립성**: 같은 데이터를 수정하지 않는 작업
3. **로직 독립성**: 서로의 결과에 의존하지 않는 작업

### 병렬 불가 조건

1. **순차 의존성**: 이전 작업의 결과가 필요한 경우
2. **리소스 충돌**: 같은 파일이나 데이터를 수정하는 경우
3. **통합 작업**: 여러 결과를 합치는 작업

## 주요 특징

- **에이전트 실행 최적화**: 병렬 처리로 전체 작업 시간 단축
- **명확한 의존성 표시**: 작업 순서와 병렬 가능 여부 명시
- **실행 가능한 명령어**: 에이전트 실행 예시 포함
- **충돌 방지**: 파일 및 리소스 충돌 사전 체크

## 사용 방법

1. `/tasks /prd/[feature_name].md` 명령어 실행
2. 자동으로 하위 작업 생성 및 병렬 처리 가능 작업 식별
3. 생성된 작업 목록을 `/tasks/` 디렉토리에서 확인
4. 🔄 표시된 작업은 병렬로 실행하여 시간 단축
